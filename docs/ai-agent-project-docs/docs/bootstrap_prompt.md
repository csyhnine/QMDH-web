# Bootstrap Prompt

你现在不是单纯写代码，而是在参与一个“多 agent 接力开发”的项目。  
这个项目的核心原则是：

1. 仓库是唯一长期记忆载体  
2. 文档定义协作协议  
3. 任意账号都只是可替换执行 agent  
4. 所有 agent 必须通过 GitHub 同步状态，而不是依赖聊天上下文

你的任务不是直接开始随意开发，而是先把当前项目整理成适合多 agent 持续接手的结构，并明确后续协作规则。

请按下面目标执行：

## 一、先检查并调整项目架构
把项目整理成适合多 agent 协作的结构。优先检查是否具备以下内容：

- `docs/plan.md`
- `docs/architecture.md`
- `docs/protocol.md`
- `docs/tasks.md`
- `docs/handoff.md`
- `docs/decisions.md`
- `docs/review.md`

如果现有仓库没有这些文件，请结合项目现状创建初版；如果已经有类似内容，请整合、去重、统一命名。

## 二、建立多 agent 协作分工规则
请把项目中的 agent 分工机制写清楚，并和当前项目实际情况匹配。

## 三、建立 GitHub 同步规则
请定义清晰的同步机制，确保不同账号切换后能继续开发。

## 四、输出形式要求
请直接产出以下结果：
1. 建议后的项目目录结构
2. 每个核心文档应包含的内容
3. 多 agent 分工设计
4. GitHub 同步和交接流程
5. 如有必要，直接生成这些 md 文件的初版内容

## 五、执行约束
- 不要脱离现有项目随意设计过度复杂的系统
- 优先复用当前仓库已有内容
- 只做能支持长期多 agent 接手的最小必要改造
- 输出要能直接落地，而不是概念讨论
