# 平台总 PRD

## 1. 产品定位

QMDH Web 是设计院内部 AI 升级平台，一期目标不是做对外商业化 SaaS，而是统一接入多个闭源模型平台，形成可复用的图像、视频、工作流与图库沉淀能力。

## 2. 产品目标

- 为设计人员提供统一入口，减少在多个 AI 平台之间切换
- 用模块化页面承载不同能力，而不是把所有功能堆在一个工作台
- 把优质结果、提示词和标签沉淀到图库中心，支持内部复用
- 保持前台聚焦生产，不暴露后台审计与成本视图

## 3. 一期范围

### 包含

- 首页模块门户
- 图像生成模块
- 视频生成模块
- 工作流模块
- 图库中心

### 不包含

- 前台审计与成本页面
- 多角色后台管理系统
- 对外开放平台与租户能力
- 自研基础模型

## 4. 用户角色

- 设计师：发起图像与视频任务，复用图库内容
- 主创建筑师：浏览工作流，沉淀优质案例
- 审核人：通过项目状态文件和模块 PRD 跟踪进度

## 5. 核心模块

## 5.1 首页

- 简要介绍平台能力
- 展示图像生成、视频生成、工作流、图库中心四个模块入口
- 展示少量平台近期动态

## 5.2 图像生成模块

- 发起效果图和局部改图任务
- 自动筛选兼容模型供应商
- 展示最近图像任务结果

## 5.3 视频生成模块

- 发起漫游视频、演示片和封面动画任务
- 展示最近视频任务与视频资产

## 5.4 工作流模块

- 解释当前平台支持的工作流
- 展示工作流适用场景、能力类型和兼容模型

## 5.5 图库中心

- 沉淀优秀图片与对应提示词
- 支持点赞
- 支持分享
- 按标签复用优质画面语言

## 6. 数据边界

- 前台只展示设计生产相关数据
- 审计、成本、供应商治理属于后台管理范畴
- 图库中心允许展示提示词、标签、点赞数、分享数

## 7. 一期验收

- 首页与四个模块页可正常访问
- 图像和视频任务可成功提交
- 工作流模块可展示当前工作流目录
- 图库中心可展示示例图片、提示词、点赞和分享
- 每个项目具备单独的状态文件与里程碑记录
