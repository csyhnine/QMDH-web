# Decisions

## Purpose
本文件只记录**顶层技术决策**、**重要约束**、**不可逆选择**。  
不要把琐碎实现细节写进来。  
目标是防止不同 agent 在核心问题上反复横跳。

---

## Decision Template

### Decision: [short-title]
- 状态: Proposed / Accepted / Deprecated / Replaced
- 日期: YYYY-MM-DD
- 背景:
  - 为什么需要这个决策
- 决策内容:
  - 具体决定了什么
- 影响:
  - 会影响哪些模块 / 文档 / 工作流
- 禁止事项:
  - 哪些替代方案当前不要再重复讨论
- 如被替换:
  - 被哪个决策替换

## Active Decisions

### Decision: 仓库文档而非聊天记录作为长期上下文
- 状态: Accepted
- 日期: 2026-04-17
- 背景:
  - 多账号、多会话开发时，聊天上下文不可稳定继承
- 决策内容:
  - 项目连续性依赖仓库、Git 历史和 `docs/` 文档
- 影响:
  - 所有 agent 都必须通过文档接手项目
- 禁止事项:
  - 不允许把聊天内容当作唯一权威源

### Decision: 任务必须细化到可单次提交完成
- 状态: Accepted
- 日期: 2026-04-17
- 背景:
  - 宏任务会让 agent 自行脑补范围，导致失控
- 决策内容:
  - `tasks.md` 中的任务默认要细到一次 commit 或一组紧邻 commits 可完成
- 影响:
  - 任务拆分、review、handoff 都必须围绕小任务设计
- 禁止事项:
  - 不允许用“重构用户系统”之类的模糊任务直接开工

### Decision: 不允许 force push 作为常规协作手段
- 状态: Accepted
- 日期: 2026-04-17
- 背景:
  - 多 agent 并行时，force push 容易覆盖他人状态并破坏可审计性
- 决策内容:
  - 冲突时默认停止并记录，不以强推解决协作问题
- 影响:
  - 分支策略、handoff、review 都会受到约束
- 禁止事项:
  - 非紧急且未审查情况下，不允许强推

### Decision: 核心文档必须保持精简并定期归档
- 状态: Accepted
- 日期: 2026-04-17
- 背景:
  - AI agent 的上下文窗口有限，文档无限膨胀会直接损害执行能力
- 决策内容:
  - `tasks.md`、`handoff.md`、`decisions.md` 只保留近期必要信息，历史内容归档
- 影响:
  - 接手流程、review 节奏、文档维护方式
- 禁止事项:
  - 不允许把全部历史堆在核心文档中
